User Tools

You can define a number of user tools which can be executed with a number of internal variables as parameters. User tools are useful for starting an editor, checkin/checkout files from a version control system, etc.

To define a new user tool you have to

  1. Open the user tools dialog (menu Tools/Configure/User Tools).
  2. Press the New button to add a new user tool
  3. Fill in the required fields for your user tool. As parameters you can use the provided internal variables.

The user tool will appear in the tools menu. You can add a shortcut key for the user tool (menu Tools/Configure/Keyboard).

When executing a user tool the internal variables will expand to the current values. For example using $RFilePath (right selected file path) in a command while the compare directory view is active, $RFilePath will expand to a list containing the file pathnames of the selected files from the right directory. Depending on the specified options when the command was defined, the command will be executed once with the list as an argument or once for each argument.

Example:

Command: “edit $RFilePath”
Selected files: file1.txt file3.txt file4.txt
Right directory: c:\DirRight

If “Execute once for the argument list” was selected, the command will expand to:
    edit c:\DirRight\file1.txt c:\DirRight\file3.txt c:\DirRight\file4.txt

otherwise three commands will be executed:

edit c:\DirRight\file1.txt
edit c:\DirRight\file3.txt
edit c:\DirRight\file4.txt

If the command is a console application it is sometimes useful to execute the command once for each argument, but still we don‚t want to open each time a new command window. For this case you can check the “Create batch file” check box, which will create a batch file before the command will be executed, and execute the batch file instead of the commands. Anyhow, if you are not sure that your command works well, you could check the “Create batch file only, do not execute” option and inspect the created batch file afterwards.
